What is a Full Stack Developer?





A Full Stack Developer is someone who builds every aspect of the web application - from the visuals and interactions, to the data transfer and processing.

They primarily work with two key components of web development.
There is frontend development where the developer builds the visuals displayed on the user's end.
There is also backend development where the data and logic stored on the server.



Navigating Around Git

What is Git?



The Basics :

Git is a Version Control System that allows for distribution.


1. It keeps track of the past versions of the file you may be working on in a systematic and organized fashion.

2. It allows for you to share these files to other people and they can keep track of their own past versions of the file.

3. It easily allows you to merge intro new versions of the file or revert back to previous versions of the file.

The Three States :

   1. Working Copy:
       The Working Copy is the version of the file that is currently on your disk for you to modify.
   2. Staging Area:
       The Staging Area is a file that stores information about what changes will go in your next commit.
   3. Repository:
       The Repository Stage is where the files and their snapshots are permanently stored.

Terminology :

  • master - the repository's main branch. Depending on the work flow it is the one people work on or the one where the integration happens
  • clone - copies an existing git repository, normally from some remote location to your local environment
  • commit - submitting the local files to the repository
  • fetch or pull - fetch gets the latest code from the remote repository. Pull also gets latest code from a remote repo as well as performs the merging.
  • push - to submit the code to a remote repository
  • remote - these are the remote locations of your repository
  • SHA - every commit or node in git is identified by a unique SHA key. You can use them in various commands in order to manipulate a specific node.
  • head - is a reference to the node to which our working space of the repository currently points.
  • branch - a particular label on a given node.
